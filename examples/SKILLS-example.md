# ⚡ SKILLS — Example (Sharefully)

> This is a filled-in example. A real SKILLS page starts with just the four starter skills and grows from there via `build-skill`.

---

This is the skills library. One skill per subpage.

## How to use this

- Pick the skill that matches the trigger phrase or request type.
- Execute the steps in order.
- Follow any output contract specified on the skill page.

## Skill safety classification

| Icon | Tier | Meaning |
|------|------|---------|
| ✅ | Read-only | Safe to run anytime. Makes no changes anywhere. |
| ✏️ | State-modifying | Creates, updates, or deletes content in Notion, MEMORY, or connected tools. |
| ⚠️ | Confirm before running | Irreversible or broad effects. Requires explicit human confirmation before executing. |

## Skills

| Skill | Tier | What it does |
|-------|------|-------------|
| `orient-me` | ✅ | Situation report — what's happening, what's next, what needs attention |
| `file-this` | ✏️ | Routes and records a decision, doc, or completed task |
| `wrap-up` | ⚠️ | Closes out a session and updates MEMORY |
| `build-skill` | ✏️ | Interactively builds and creates a new skill page |
| `plan-cycle` | ✏️ | Plans the next sprint cycle from Linear backlog |
| `add-issue` | ✏️ | Creates a new Linear issue from a description |
| `review-ready` | ✅ | Checks whether a branch/issue is ready for review |
| `sprint-review` | ⚠️ | Runs end-of-sprint review across Linear and MEMORY |
| `sprint-retro` | ⚠️ | Runs a retrospective and files learnings |
| `cleanup` | ⚠️ | Cleans up stale branches, issues, and MEMORY entries |
| `agent-handoff` | ✏️ | Prepares a handoff summary for the next agent session |
| `belief-consistency` | ✅ | Checks whether a doc contradicts Core Beliefs |
| `comment-handling` | ✏️ | Resolves or files inline Notion comments |
| `route-work` | ✏️ | Applies the routing rule to a piece of work and sends it to the right place |
| `multi-agent delivery` | ⚠️ | Coordinates work across multiple agents with handoff protocols |
